**Tema:** Refinamiento de Seguridad (4 Capas) y CSS

En esta bitácora se detalla el trabajo realizado para cumplir con los estándares de seguridad exigidos y el pulido visual:

1. Capa de Base de Datos: Se agregaron RegexValidator y MinValueValidator a los modelos de Usuario y Producto.
2. Capa Servidor: Se crearon métodos 'clean_' en forms.py para garantizar la sanitización y el formateo de datos (ej. evitar HTML malicioso en descripciones).
3. Capas Frontend (HTML y JS): Se agregaron atributos 'pattern' a los inputs y lógica JS antes del submit para alertar al usuario antes de tocar el servidor.
4. Mejoras Visuales: Se ajustó el archivo custom.css, corrigiendo bordes superpuestos en la barra de búsqueda y optimizando las proporciones (object-fit: contain) de las imágenes.
5. Se unificó el diseño de notificaciones con Toast UI de SweetAlert2.

El sistema es ahora altamente robusto contra inyecciones e inserciones de datos inválidos.
