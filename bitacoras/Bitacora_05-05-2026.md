**Fecha:** 05 de Mayo de 2026
**Tema:** Configuración Inicial del Proyecto Treddy

En la sesión de hoy se comenzó con la planificación y la configuración base del proyecto Treddy-Django. Las actividades principales fueron:

1. Inicialización del entorno virtual y definición de las dependencias requeridas en Django.
2. Creación del modelo personalizado de usuarios (CustomUser) heredando de AbstractUser, para habilitar los roles de Administrador, Vendedor y Cliente.
3. Configuración de la base de datos local SQLite y ejecución de las primeras migraciones.
4. Estructuración del patrón MVT (Model-View-Template) creando la aplicación 'usuarios' y definiendo las rutas básicas.
5. Integración de Bootstrap 5 de manera local para el maquetado inicial, sin depender de CDNs externos.

El objetivo de esta sesión se cumplió a cabalidad, dejando lista la arquitectura de autenticación que es la piedra angular del sistema. Se presentaron algunos retos al extender AbstractUser, pero se resolvieron configurando correctamente AUTH_USER_MODEL en settings.py.
